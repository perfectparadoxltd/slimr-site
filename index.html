// Questions
Does anything leave my Mac?

No. All compression, PDF operations and OCR run entirely on your Mac. Your files, images and PDFs are never uploaded anywhere. The only network connections Slimr ever makes are an update check and a one-off licence validation — never your documents. You can confirm this yourself with a firewall tool like Little Snitch or LuLu.

Does it work without an internet connection?

Yes — completely. Once activated, Slimr does everything offline. Compression, merge/split/extract and OCR all use frameworks built into macOS, so you never need a connection to get work done.

Is it on the Mac App Store / sandboxed?

No. Slimr is sold directly and is distributed as a Developer ID app, notarized by Apple. That means it can work with whole folders and write results beside your files without the App Store sandbox getting in the way — while still being Gatekeeper-checked and safe to run.

How many Macs can I use it on?

All the Macs you personally own, with a single licence. One purchase covers your whole setup — no per-seat counting.

What formats does it handle?

Images: PNG, JPEG, HEIC and TIFF. Documents: PDF. You can compress any of these to a target size, run PDF merge/split/extract, and OCR images or PDFs into searchable PDFs.

Will it reduce quality?

You're in control. Slimr compresses toward the target size you choose and stops there, balancing quality against size. It never up-scales or enlarges a file — if something is already under the target, it's left as-is.

Does the mirror folder touch my originals?

Never. When you compress a folder, Slimr writes the results to a brand-new folder named "<name> — Slimr" right beside the source. Your originals are read but never modified, moved or deleted.

What languages does the OCR support?

OCR currently recognises Spanish and English text, using Apple's on-device Vision engine. The recognised text is added as an invisible, selectable layer so the page looks unchanged but becomes searchable.
